# S5. 외부 기획안 모드

내부 파이프라인이 아닌 외부 클라이언트를 위한 기획안 작성 워크플로우. 리서치에서 기획안까지 3단계로 진행한다.

**스코프 한정 (순환 방지):** 정부·지자체·선거 후보 대상 정책기획은 **policy-planning 관할**이므로 S5로 처리하지 않는다. S5는 비정책 외부 기획안(기업 컨설팅·관광공사·기관 등)에만 적용. policy-planning이 내부 로드맵 필요 시 이 스킬을 하위 호출할 수 있으나, **이 스킬에서 policy-planning을 역호출하지 않는다.**

## 5a. 프레임 설계 (형 확인 게이트)

과제를 받으면 곧바로 조사하지 않고, 먼저 리서치 축·가설·질문을 설계한다. 축 없이 조사하면 수집 데이터가 산만해지기 때문이다.

**절차:** ①과제 분해(수신자·목적·제약 파악) → ②축 정의(기본렌즈6 기반, 과제별 가감) → ③축별 가설+질문 목록 → ④**형 확인** 후 조사 착수

**예시:**
```
과제: 인천 문화관광 전략 기획안
수신자: 인천관광공사
축: 관광자원현황 / 타깃분석 / 경쟁벤치마크 / 정책환경 / 경제성 / 콘텐츠전략 ...
가설: "인천의 문화관광 자원은 서울 근접성 대비 과소평가"
질문: "인천 방문 관광객 중 문화목적 비율은?"
→ 형 확인 → 축 조정 → 조사 착수
```

## 5b. 축별 조사

축당 독립 리서치. UP E절 수렴검증4축 적용. 종합보고서는 `_research/` 폴더에 저장.

**캐스케이드 연결:** 내부 파이프라인의 갭 탐지가 축별 조사에 대응. 축 = 분석 영역, 갭 = 각 축에서 발견된 과제.

## 5c. 스파인 도출 → 기획안

축간 교차분석으로 핵심 통찰·전략 2~3개를 도출한다. 이것이 기획안의 스파인이 된다.

**절차:** ①축별 핵심 발견 요약 → ②축간 교차점 식별 → ③통찰→전략 변환(2~3개) → ④스파인 확정 → ⑤기획안 작성(C절 모래시계 구조)

**기획안 구조:** 스파인→본문(핵심만)→부록(접힘). 수신자 맞춤. 기본=마크다운 포맷(콜아웃·테이블·헤딩), 애플디자인 모드시 apple-design-style HTML div 박스, div 스타일 지정시 html-div-style HTML div 박스 레이아웃. C절 모래시계 구조 + H-1 마크다운 기본 포맷 적용.
