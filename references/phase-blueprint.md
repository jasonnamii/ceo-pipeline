# S1. 청사진 — 도달할 모습

| 항목 | 내용 |
|------|------|
| 성격 | 비전문/미션문. 구체적이되 실행 수준은 아님 |
| 시야 | 6개월~1년 |
| 질문 | "우리 제품/회사가 [시점]에 어떤 모습이어야 하는가?" |
| 갱신 | 상시. 큰 방향 변경 시 Major up |

**포맷:** 도달할 모습(2~5문장) → 이것이 맞다고 믿는 이유 → 이것이 아닌 것(범위 한정 필수). 가설 검증 프레임 강제 않되, 근거는 명시.

**캐스케이드 연결:** 청사진의 "도달할 모습"이 모호하면 갭 탐지가 작동하지 않는다. "상태"로 기술해야 비교 가능하기 때문이다.
